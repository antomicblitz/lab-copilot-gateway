# T5 Handoff — Protocol Lookup Service

**Slice 7:** Protocol lookup service that searches approved eLabFTW protocol/resource entries before recommending PCR/assembly conditions.

## What Was Built

### 1. `src/lab_copilot_gateway/protocol_lookup.py` (new)

- `ProtocolLookupService` — searches eLabFTW items for approved protocol entries matching `method_type`, `reagent_product_name`, and `aliases`.
- `ProtocolMatch` / `ProtocolLookupResult` dataclasses with `to_dict()` for API serialization.
- Matches via hierarchy: exact reagent name → alias → method-only → fallback.
- `validate_corpus()` — validates protocol entries for missing fields, duplicate aliases, deprecated entries.
- Only selects `status == "approved"` AND `approved_for_use == True` entries. Deprecated entries are excluded from selection but flagged during validation.
- Falls back to `fallback_used=True` + warnings when no approved protocol is found.

### 2. Tool registration (`tools.py`)

Added 2 tools to the curated catalog:

- `opencloning.lookup_protocol` (tier: VALIDATION_DRY_RUN, read-only) — search eLabFTW for approved PCR/assembly protocol conditions.
- `protocols.validate_corpus` (tier: VALIDATION_DRY_RUN, read-only) — validate protocol corpus integrity.

Both use `adapter="opencloning"` so they route through the opencloning dispatch path.

### 3. Dispatch wiring (`app.py`)

- Imported `ProtocolLookupService`.
- Added dispatch cases in `_invoke_opencloning_tool()` for both tools.
- Added `_invoke_opencloning_lookup_protocol()` and `_invoke_opencloning_validate_corpus()` handler functions.
- Uses `adapter.elabftw_client` to access the eLabFTW client from the OpenCloning adapter singleton.

### 4. Tests (`tests/test_protocol_lookup.py`)

23 tests covering:

- Exact reagent match returns full protocol fields (Phusion PCR, NEBuilder Gibson).
- Alias match (reagent name matches an alias, catalog number, etc.).
- Method-only match (no reagent name specified, returns first approved entry).
- Fallback when no protocol found → `fallback_used=True` with warnings.
- Deprecated entries are excluded from selection (but flagged in validation).
- `validate_corpus` detects: missing fields, duplicate aliases, deprecated entries, non-approved warnings.
- Edge cases: empty method_type, empty category, items without metadata.

## Key Design Decisions

- Uses `adapter.elabftw_client` from the OpenCloning adapter singleton (avoids a separate singleton or factory).
- Post-filters by exact `method_type` from metadata, because `search_items` does free-text matching that can return collateral matches (e.g. "ligation" matches "restriction_ligation").
- Post-filters by `method_type` from extra_fields; items without `method_type` in extra_fields are skipped.
- `validate_corpus` has a fallback path for items with extra_fields but missing required fields (detects them even when `_parse_item` returns None).
