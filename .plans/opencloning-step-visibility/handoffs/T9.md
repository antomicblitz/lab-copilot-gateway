## Summary

Built a deterministic validation bundle that gates the writeback approval flow.
The bundle is constructed from captured cloning run facts (sequences, sources,
primers, artifacts, step manifests) â€” never from LLM output â€” so the LLM
cannot influence validation results. Structural blockers prevent approval;
warnings are visible but non-blocking.

## Key Design Decisions

- **Blockers are absolute**: `finalize()` sets `is_approved = False` if any
  blocker check has status `fail`. No override path exists in the code.
- **Warnings are non-blocking**: warnings are collected for display but never
  affect `is_approved`.
- **Deterministic**: the bundle is built from captured facts only. The LLM
  cannot influence which checks pass or fail â€” it can only affect the cloning
  run state that the checks examine.
- **Final construct detection**: a leaf sequence (id not referenced as an input
  by any source) is the final construct. Falls back to the last sequence with
  `file_content` if no leaf exists.
- **GenBank artifact detection**: accepts either the normalized artifacts
  payload (with an `artifacts` list) or `approval_args` (with
  `artifact_content` or `artifact_id`), covering both inline and bundle-ref
  writeback paths.
- **Annotation guard formalized**: the existing `_maybe_rewrite_genbank_features`
  best-effort re-annotation is reflected as the `post_assembly_annotation`
  check. Primary signal is step manifests; fallback is sequence features.
- **Primer details**: "details" means analysis fields (Tm, GC content,
  heterodimer), not the primer sequence itself â€” the sequence is the primer,
  not a detail about it.
- **Notebook/audit drafts**: always warn as `not_checked` until T10 implements
  draft generation. The check methods accept a `draft_generated` boolean so T10
  can flip them to `pass` without changing the bundle structure.
- **Integration point**: the bundle is built inside `writeback_artifact()`
  before the upload and included in the result dict. `app.py` surfaces it at
  the top level of the dispatch response. For now, the bundle reports status;
  the widget/orchestrator enforces the gate using `is_approved`.

## Open Risks

- The bundle is currently built during `writeback_artifact` (after approval is
  consumed). Full enforcement (blocking approval BEFORE it is requested)
  requires a separate pre-approval validation endpoint â€” deferred to a later
  slice. For now, the bundle reports status; the widget uses `is_approved` to
  gate the approval UI.
- Step manifests are not yet wired into the adapter's dispatch path (T2/T6
  work). The `step_manifests` parameter defaults to empty; annotation and
  heterodimer checks fall back to heuristic detection from sequence features
  and primer data. Once T2/T6 wire step manifests, the checks will use the
  primary signal automatically.
