# T13 Handoff — E2E Test Fixtures and Regression Coverage

**Slice 12** of the OpenCloning Step Visibility build plan.

## Deliverables

### New file: `tests/test_opencloning_e2e.py`

5 integration/E2E tests that verify the full end-to-end flow of multiple slices working together:

1. **`test_e2e_happy_path_gibson_workflow`** — Complete Gibson cloning workflow:
   - Simulates 6 steps: sequence import × 2, primer design, PCR, Gibson assembly, annotation
   - Verifies step manifests are typed correctly (operation_type chain, parent_step_ids)
   - Verifies validation bundle passes (all 10 checks, `is_approved=True`)
   - Verifies notebook summary contains all sections (objective, method, primers, PCR products, protocol conditions, final construct, validation status, attachments, AI rationale)
   - Verifies audit report contains all sections (step timeline, intermediate products, primers, protocol, validation, provenance, attachments)
   - Verifies writeback succeeds with all upload IDs and package hash

2. **`test_e2e_region_correction_invalidates_hash`** — Correction scenario:
   - Builds manifests with region `(100, 1300)`, corrects to `(200, 1100)`
   - Verifies old region refs differ from new
   - Verifies both pass validation (region is not a blocker)
   - Verifies the serialized manifests produce different hashes (correlation: different manifests → different state)

3. **`test_e2e_protocol_fallback_flows_through_all_layers`** — Missing protocol fallback:
   - Protocol lookup returns `fallback_used=True` with warning
   - Verification bundle: protocol_lookup check status is `fail`, included in warnings
   - Notebook summary: validation section reflects the warning count
   - Audit report: includes "## Fallback Assumptions" section with the warning message

4. **`test_e2e_preview_store_lifecycle`** — Full preview lifecycle via HTTP:
   - Stores 2 previews
   - Fetches with correct token → 200 with correct payload shape (ref, run_id, step_id, sequence_id, file_format, file_content, is_circular, sequence_length)
   - Fetches with wrong token → 404
   - Fetches missing ref → 404
   - Fetches expired preview → 404

5. **`test_e2e_partial_writeback_failure`** — Partial writeback:
   - GenBank upload succeeds (via stub)
   - History JSON upload is skipped (no strategy data)
   - Audit report upload succeeds
   - Notebook append fails (patched stub raises RuntimeError)
   - Result status is "partial" with rollback instructions
   - Each step's individual status is verified

## Key Design Decisions

- **Realistic mock data**: Uses GenBank fixture sequences matching the patterns from T1/T9/T10 tests (LOCUS lines, FEATURES, ORIGIN, `//` termination).
- **Real step manifests via `extract_step_manifest()`**: Manifests are built using the same dispatcher that the production path uses, ensuring the test exercises actual extraction logic rather than hardcoded dicts.
- **Cross-slice verification**: Each test checks at least 3 layers (e.g. manifests → validation → reports → writeback) to confirm integration.
- **Reuses shared fixtures**: `token`, `identity`, `elabftw_stub`, `approval_store`, `writeback_adapter`, and `client` from the existing test patterns.
- **Complexity ≤ 15**: Each test function is a single assertion block with clear sections.
