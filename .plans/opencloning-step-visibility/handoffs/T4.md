# T4 Handoff — Short-lived preview store and fetch endpoint

**Slice 4** of the OpenCloning Step Visibility build plan.

## Deliverables

### New file: `src/lab_copilot_gateway/opencloning_previews.py`

- `OpenCloningPreviewStore` — in-memory, context-bound, TTL-expiring store for
  intermediate sequence payloads (GenBank/Fasta).
- `PreviewPayload` dataclass carrying `preview_ref`, `run_id`, `step_id`,
  `context_token_hash`, `sequence_id`, `file_format`, `file_content`,
  `sequence_length`, `is_circular`, timestamps.
- Default TTL: 30 minutes. Max 50 previews per run.
- `get_preview_store()` / `reset_preview_store()` singleton pattern (matches
  `ArtifactBundleStore` convention).

### Modified: `src/lab_copilot_gateway/opencloning.py` (adapter)

- `OpenCloningAdapter.capture_preview_refs()` — new public method that iterates
  `result["sequences"]`, stores each with `file_content` in the preview store,
  and returns `{seq_id: preview_ref}` mapping. Added after `call_endpoint()`
  (before `writeback_artifact` section).

### Modified: `src/lab_copilot_gateway/app.py`

- `_opencloning_invoke_success()` — captures preview refs after each tool call
  by calling `adapter.capture_preview_refs()` with the raw result. Computes
  `token_hash` from the context token for auth binding. Injects `preview_ref`
  into each sequence in the redacted response so the LLM/widget can pass it
  to the fetch endpoint.
- `_capture_opencloning_preview_refs()` — helper that delegates to the adapter.
- `_inject_preview_refs_into_result()` — recursive helper that mutates a
  result dict to carry `preview_ref` on each sequence.
- `_register_opencloning_preview_routes()` — registers
  `GET /v1/opencloning/previews/{preview_ref}` with context-token auth.
  Returns 404 on missing, expired, or token-mismatched previews.
- Route registered in `create_app()`.

### New file: `tests/test_opencloning_previews.py`

14 tests covering:

- Store and fetch round-trip
- Wrong context token → None
- Expired payload → None
- Missing preview_ref → None
- Per-run limit eviction (oldest removed)
- Empty file_content stored with length 0
- Multiple runs independent
- `capture_preview_refs` produces correct mapping
- `capture_preview_refs` with empty/no sequences → empty dict
- HTTP endpoint: 200 success with correct payload shape
- HTTP endpoint: 404 for missing ref, wrong token, expired
- HTTP endpoint: 422 when token header missing

## Integration Point

Preview refs are captured in `_opencloning_invoke_success()` (app.py) after
each tool call that returns a result. The raw `result.result` dict is examined
for sequences with `file_content`. The `preview_ref` is injected into each
sequence's dict in the redacted response under the key `"preview_ref"`.

## Design Decisions

- **No `run_id`/`step_id` in adapter methods** — these aren't threaded through
  the adapter's method signatures. Instead, `plan_id` (derived from `request_id`)
  serves as `run_id`, and `audit_action_id` serves as `step_id` at the
  `_opencloning_invoke_success` level.
- **Preview refs survive redaction** — `_inject_preview_refs_into_result`
  injects `preview_ref` into the sequences AFTER `_redact_opencloning_file_content`
  so the widget/LLM sees them alongside the redacted `file_content` metadata.
- **In-memory only** — previews are lost on gateway restart (acceptable for V1).
  Future slices can add a SQLite or Redis backend if persistence is needed.

## Next Slices Expect

- T5/T6 will wire `preview_ref` into `StepManifest` objects created by
  `extract_step_manifest()` — the `preview_ref` field already exists on the
  dataclass (default: `None`), and `capture_preview_refs()` provides the
