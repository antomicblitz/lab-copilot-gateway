## Decisions Made
- **Extractor dispatch** maps endpoint paths via `_ENDPOINT_ROUTING` exact match or prefix match, with generic fallback. This matches the plan's "dispatcher maps endpoint paths to extractors."
- **Primer Tm mismatch warning** added as a heuristic (ΔTm > 5°C generates a warning). This is a reasonable quality gate for primer design steps.
- **Feature-loss warning** from the `/pcr` endpoint's `feature_loss_warning` dict is forwarded verbatim into the manifest warnings list.
- **Restriction enzymes** extracted from `RestrictionSource` metadata when available; warns if no enzymes detected.
- **Placeholder extractors** (`protocol_lookup`, `validation`, `writeback_draft`) are stubs that emit valid manifests with empty refs. They accept the same signature as real extractors so the dispatcher can route to them without special-casing.
- `extract_step_manifest()` sets `schema`, `run_id`, `step_id`, `parent_step_ids` on the already-populated manifest rather than threading them through each extractor.
- `to_dict()` returns explicit snake_case keys matching the plan's JSON schema (not dataclass `asdict()`), ensuring full control over serialized shape.
